If you work primarily in the JavaScript ecosystem, .NET might feel like legacy software. Something your enterprise clients use. Something you inherit, not choose.

That's a misread of where .NET is today.

## What changed

.NET Core (now just .NET) rewrote the story. Cross-platform, open source, faster than most alternatives on benchmarks that matter. ASP.NET Core consistently sits near the top of TechEmpower plaintext and JSON benchmarks. The runtime has had years of performance investment.

The tooling caught up too. Hot reload works. The SDK is clean. NuGet is a real package manager. If you tried .NET before 2017 and wrote it off, you're judging a different product.

## What it's actually good at

Backend APIs. This is where .NET earns its keep. The combination of C#'s type system, ASP.NET's routing and middleware model, and Entity Framework's migrations makes CRUD-heavy APIs fast to write and easy to maintain.

`async/await` in C# predates most other languages. The `Task` model is mature. I've never had the "did I block the thread?" anxiety in .NET that I used to have in earlier async Python or early Node.

The dependency injection system built into ASP.NET is one of the best in any framework. It's opinionated enough to prevent the wrong patterns and flexible enough to accommodate everything I've needed.

## What I use it for

Internal tooling and APIs where the team has C# familiarity. Data pipelines where strong typing catches schema issues at compile time instead of runtime. Anything that integrates with Windows infrastructure — Active Directory, COM, legacy SQL Server setups.

For greenfield public APIs where I'm choosing freely, I might reach for Python for speed of iteration or Go for deployment simplicity. But .NET is never the wrong choice for backend work. It's solid in a way that doesn't make the news.

## The honest criticism

The ecosystem is smaller than JavaScript's for certain domains. Frontend .NET (Blazor) is interesting but I wouldn't choose it over React for most projects. The enterprise reputation isn't completely unearned — some of the older patterns in ASP.NET MVC are genuinely verbose.

But for the backend, the criticism mostly evaporated post-.NET Core. What's left is aesthetic preference, not engineering reality.

Reach for it without embarrassment.
